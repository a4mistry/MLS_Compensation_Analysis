# CLAUDE.md — MLS 2026 Compensation Analysis

Guide for an AI agent picking up this project. Read this first.

## What this is

A data-journalism project analyzing **Major League Soccer 2026 player compensation** and how
spending translates into on-field results. It produces:

1. A reproducible **Python pipeline** (PDF → clean data → analysis → charts → website data).
2. An interactive **scrollytelling website** (`web/`) — dark-to-light "pitch" themed, built with
   ECharts, deployed to **GitHub Pages**.
3. Standalone **experiments** (`experiments/`) for deeper questions not on the main site.

**Live site:** https://a4mistry.github.io/MLS_Compensation_Analysis/
**Repo:** https://github.com/a4mistry/MLS_Compensation_Analysis (branch `main`)

## The one metric to know

Everything headline uses **`guaranteed_comp`** (guaranteed compensation = base salary + marketing/bonus
money), the standard MLS pay figure. `base_salary` is also in the data (used only by the "marketability
gap" analysis). A fake club **"MLS Pool"** exists in the salary data and **must be excluded** from all
club-level analysis (every script does `players = sal[sal["club"] != "MLS Pool"]`).

## How to run

```bash
python main.py          # full pipeline: extract -> fetch -> analysis -> efficiency -> export
```
Then open `web/index.html`. To update the live site: `python main.py`, commit, push (the Action redeploys).

- `fetch_standings.py` pulls **live** standings from ESPN each run, so results refresh; salaries are a
  fixed PDF snapshot. Override season with `MLS_SEASON=2027 python main.py` (see open item M7 below).
- Individual scripts are runnable standalone (`python code/export_web_data.py`); each resolves paths from
  the project root via `Path(__file__)`, so cwd doesn't matter.

## Environment / conventions (important gotchas)

- **Windows + Git Bash.** `python` is Python 3.14 (pythoncore). Historically `pip` resolved to a *different*
  interpreter — **always use `python -m pip install ...`**, never bare `pip`.
- Deps: `pandas numpy scipy matplotlib seaborn pdfplumber` (+ ECharts/Google Fonts from CDN in the browser).
- **The website is a static bundle.** `web/site_data.js` is committed so Pages needs no build step. The
  browser reads `window.MLS_DATA`; opening `web/index.html` from disk works (no server needed).
- **Data-bound copy pattern (critical).** A council audit found hardcoded numbers in the HTML drift out of
  sync with the data. The fix pattern, now used throughout: put an empty `<span id="foo"></span>` in
  `index.html` and populate it in `app.js` from `window.MLS_DATA`. **When adding/altering any number the
  site displays, bind it — do not hardcode.** Search `app.js` for `putText(` to see all bound spans
  (Atlanta paragraph, cost-per-point spread, footer season-progress + "as of" date, etc.).
- **Position bucketing is centralized** in `code/positions.py::pos_bucket` (Attack/Midfield/Defense/
  Goalkeeper). It prints a loud **terminal alert (stderr)** the first time it sees an unrecognized
  position, then defaults to Midfield. Both the pipeline and `experiments/` import it — do not re-define it.
- **Non-finite guard.** `export_web_data.py` guards `pts==0 / gp==0` divisions and runs all floats through
  `_num()` so `site_data.js` can never contain `Infinity`/`NaN` (which would be invalid JSON). Keep this.

## Verifying visual changes (no test suite)

There are no automated tests. To verify the site after changes, render it headless with Edge and read the
PNG (this is how all prior visual work was checked):

```bash
EDGE="/c/Program Files (x86)/Microsoft/Edge/Application/msedge.exe"
"$EDGE" --headless=new --disable-gpu --hide-scrollbars --window-size=1180,7400 \
  --virtual-time-budget=14000 --screenshot=out.png "file:///ABS/PATH/web/index.html"
```
Scroll-reveal sections start at `opacity:0`; to capture them in a static screenshot, inject a temp
`<style>.reveal{opacity:1!important;transform:none!important}</style>` into a throwaway copy of index.html
(the established trick — grep this file's git history / prior work). Wrap Edge calls in `timeout 90` — a
headless process occasionally hangs; kill leftovers with PowerShell `Stop-Process`.
Also run `node --check web/app.js` after editing JS.

## Data shape of `window.MLS_DATA` (what app.js consumes)

Top-level keys: `league`, `hist`, `topEarners`, `positions`, `clubs`, `regression`, `roster`, `gap`,
`goals`, `middle`, `meta`. Each maps to a website section:

| Section (index.html) | Data key | Notes |
|---|---|---|
| 01 Landscape / 02 Superstars | `league`, `hist`, `topEarners` | totals, Gini, distribution, top earners |
| 03 Market by role | `positions` | median pay by position (n>=5) |
| 04 Money vs results | `clubs`, `regression` | payroll vs PPG scatter |
| 05 Value for money | `clubs`, `league.costPerPointSpreadX` | cost-per-point |
| 06 Roster construction | `roster` | Gini scatter, top-earner positions, Atlanta spotlight |
| 07 Marketability gap | `gap` | base vs guaranteed comp |
| 08 Does spending buy goals | `goals` | attack$→GF, def$→GA, efficiency |
| 09 Stars vs middle class | `middle` | top-3 vs rest-of-roster (2 tiers; "roster floor" was intentionally dropped) |
| footer | `meta`, `league.asOf` | standings as-of date, games-played range, season stage |

## Working norms for this repo

- **Commit/push only when the user asks.** History shows the user drives this explicitly ("push to git").
- Commit-message trailers are used (Co-Authored-By + Claude-Session). Match the existing style.
- `.gitattributes` normalizes line endings to LF — the CRLF warnings on `git add` are harmless.
- Generated files (CSVs, `site_data.js`, charts, `standings_meta.json`) ARE committed intentionally so the
  static site and analysis are viewable without a rebuild.
